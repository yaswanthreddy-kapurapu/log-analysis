Portable Log Analysis for Air-Gapped Networks

A secure, offline-first log ingestion and analytics platform designed for isolated environments — portable, auditable, and deployable from USB or local media.

See Architecture Download

Key Benefits

  • Zero external dependencies — works fully offline
  • Portable deployment: USB, local server or container
  • Strong encryption, audit trails, and role-based access

Problem Statement & Objective

Organizations with air-gapped (isolated) networks face unique challenges: limited/zero connectivity, strict data exfiltration policies, and difficulty applying centralized analysis and updates. The objective is a portable, secure log analysis tool that can ingest diverse log sources, analyze and generate reports entirely offline, and provide a secure mechanism to export results for authorized transfer.

  • Challenges: offline operation, limited tooling, secure transfer.
  • Objective: provide portability, forensic integrity, and enterprise-grade security.

Solution Overview

This tool is an offline-first platform built to operate without network access. It emphasizes portability (USB-bootable or local server), minimal runtime dependencies, deterministic analysis, configurable rule sets, and strong security controls. Operators can collect logs, run local analyses, generate alerts and reports, and export sanitized artifacts for secure transfer.

Design Philosophy

  1. Offline-first — all functions run locally, no external telemetry.
  2. Portable & Deterministic — predictable behavior across environments.
  3. Secure by design — encryption, access control, signed exports.

Component Breakdown

Log Ingestion Engine

Responsible for collecting logs from removable media, syslog captures, device exports, or manual uploads. Designed for batch import and live tailing when permitted. Includes safe handling and integrity verification (checksums, signatures).

Parsing & Normalization Module

Transforms raw logs into a common schema. Uses pluggable parsers (regex, GROK-like patterns, JSON/CEF parsers) and normalization rules to map fields to canonical types for indexing and correlation.

Rule-Based & Pattern Analysis Engine

Runs deterministic pattern matching, correlation rules, and anomaly detection (statistical baselines). Rule sets are stored locally and can be versioned and signed to ensure integrity.

Storage Layer (Local/Embedded DB)

Lightweight embedded database (e.g., SQLite or LiteDB) for portability and ease of backup. Optimized for read-heavy analytic queries and supports indexing for fast search.

Visualization & Reporting Module

Local dashboards, timelines, filterable tables, and exportable PDF/CSV reports. Designed for fast, offline rendering with resource-constrained hardware in mind.

Export & Portability Module

Generates signed, optionally encrypted export bundles that include sanitized data, reports, and audit logs. Includes export signing keys and verification metadata to support secure transfer to other environments.

Workflow Diagrams

Log Collection

Offline collection is done via removable media ingestion or manual upload. The system computes and stores a checksum and ingest metadata.

Offline Analysis

Analysts run pre-configured analysis jobs, review dashboards and timelines, and annotate findings. All artifacts are stored locally and appended to the audit log.

Alert Generation

Rules trigger local alerts that are visible in the dashboard and optionally exported in signed alerts bundles for transfer.

Report Export & Secure Transfer

Users generate signed exports that include reports and minimal sanitized data. Export bundles are hashed and signed; the receiving environment verifies signatures before acceptance.

API Design (Internal & Local)

APIs are intended for local automation and integration with other offline tools. Authentication is local token-based or via HSM-protected keys where available. No external network calls are required.

Authentication

Local token or key file (signed). Example header: Authorization: Token {local_token}. For higher assurance deployments, use hardware-backed keys and signed tokens verified by the local runtime.

Audit & Record Keeping

All API operations are appended to the audit log with user, timestamp, operation, and object hashes to provide tamper-evident records.

Data Flow & Processing Logic

Logs enter via ingestion, pass through the parser & normalizer which maps to a canonical schema, then are indexed and stored. Analysis rules run over indexed data to produce alerts and aggregates. The system keeps provenance metadata for every transform to support forensic auditing.

Security & Isolation Model

  • Air-gapped operation: No outbound networking; package updates are applied via signed bundles only.
  • Access control: RBAC with local user database and optional HSM integration.
  • Encryption at rest: AES-256 for bundled exports and optional full-disk encryption for storage.
  • Audit logging: Append-only audit trail with hashes and signatures.
  • Zero external dependencies: All required binaries and rule sets are provided in signed, versioned packages.

Deployment Model

Multiple deployment options depending on operational constraints:

  • USB-bootable image: Full OS + application for rapid deployment.
  • Local server: Deployed to an isolated machine in the target network.
  • Containerized: For environments that allow local container runtimes (no external pulls).
  • Laptop-based: Analysts run the tool on secure, air-gapped laptops for field operations.

Use Cases

  • Government offices with restricted networks
  • Defense and military networks
  • Critical infrastructure and utilities
  • Disaster recovery and emergency field sites

UI / UX Design

Enterprise-grade, minimal UI with strong information density. Key features:

  • Event drill-down, context panels and timeline scrubber
  • Report builder with templating and signed export
  • High-contrast, accessible color palette and keyboard-first navigation
